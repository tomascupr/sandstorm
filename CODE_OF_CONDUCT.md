# Code of Conduct

## Our Standard

We want Sandstorm to be a practical, respectful project for users and contributors. Participate in
ways that keep discussion technical, constructive, and inclusive.

Examples of expected behavior:

- Be respectful and direct when giving feedback
- Assume good intent, but challenge ideas rigorously
- Focus on bugs, behavior, tradeoffs, and user impact
- Share context, logs, repro steps, and concrete suggestions when reporting problems

Examples of unacceptable behavior:

- Harassment, insults, threats, or intimidation
- Discriminatory language or personal attacks
- Publishing private information without permission
- Repeatedly derailing technical discussion or spamming project channels

## Scope

This code of conduct applies to GitHub issues, pull requests, discussions, and any other project
spaces where contributors represent the Sandstorm community.

## Enforcement

Project maintainers may remove comments, close threads, reject contributions, or block access when
behavior violates this policy.

If you need to report a conduct issue privately, contact a maintainer directly through GitHub
instead of opening a public thread.
